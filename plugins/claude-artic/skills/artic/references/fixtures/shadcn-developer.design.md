# shadcn/ui Developer SaaS Pattern Notes

## Reusable Patterns
- Prefer composable primitives, restrained borders, and token-driven component variants.
- Use cards, muted panels, badges, and command/code surfaces for developer-tool clarity.
- Keep implementation-friendly Tailwind naming without copying a specific template.

## Components
- Primary and secondary buttons.
- Cards with border, surface, text, and muted text roles.
- Muted panels for workflow explanations or code-adjacent context.

## Accessibility
- Ensure keyboard focus states and semantic button/link distinctions.
